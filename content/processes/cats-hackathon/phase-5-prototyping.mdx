# Phase 5: Prototyping Workshop

**Date:** December 6, 2024 (Friday)  
**Format:** In-person session at Next Trend Hub  
**Duration:** Full day (9 AM - 5 PM)

## Purpose

This is your **second in-person session** where you'll:
- Complete Quadrant 4 (Define Opportunity)
- Explore multiple solution options
- Get mentor feedback on feasibility
- Plan your build sprint
- Discuss Cardano integration possibilities

## Session Structure

### Morning (9 AM - 12 PM)
**Solution Brainstorming**
- Each team presents their hypothesis
- Brainstorm 3-5 potential solutions
- Evaluate feasibility of each option
- Create scoring matrix for comparison

### Midday (12 PM - 2 PM)
**Cardano Integration Discussion**
- Optional session on blockchain integration
- Review use cases where Cardano adds value
- Discuss technical feasibility
- Get guidance on testnet vs mainnet

[Cardano Integration Guide →](/processes/cats-hackathon/cardano-integration)

### Afternoon (2 PM - 5 PM)
**Prototype Planning**
- Select your final solution approach
- Define MVP scope for Dec 7-13 build sprint
- Create technical architecture plan
- Assign team roles for build phase
- Set daily check-in schedule

## What to Bring

✅ Team's hypothesis statement  
✅ 3-5 solution ideas (rough concepts)  
✅ Questions about technical implementation  
✅ Laptop for planning and documentation  

## Key Outputs

By end of Dec 6, each team should have:

1. **Completed Opportunity Definition** (`/team/[your-team]/define-opportunity`)
   - 3-5 solution options explored
   - Scoring matrix comparing options
   - Selected solution with justification

2. **Build Sprint Plan**
   - MVP features list (priority ordered)
   - Technical stack decisions
   - Team role assignments
   - Daily milestone targets

3. **Prototype Timeline**
   - Dec 7-9: Core functionality
   - Dec 10: First prototype submission
   - Dec 11-12: Refinement
   - Dec 13: Final presentation

## Cardano Considerations

**Good Fit If:**
- Need transparency or trust (supply chain, donations, voting)
- Handling value transfer (payments, tokens)
- Want immutable records (certifications, proofs)

**Not Necessary If:**
- Simple web app without trust issues
- Internal tool for single organization
- Time-sensitive data that needs deletion

## Next Phase

**Phase 6 (Dec 7-12):** Build Sprint - Full-time development of your solution with daily check-ins.
